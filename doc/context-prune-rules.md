# Context Prune Rules

## 목적
- 중간 컨텍스트를 최소 비용으로 유지한다.

## 보관 기준
- 제품 의사결정에 영향이 있는 항목
- 구현/테스트에 필요한 수치, 정책, 제약
- 되돌릴 수 없는 선택(네이밍, 데이터 모델, 결제 정책)

## 삭제 또는 이관 기준
- 중복 메모
- 이미 해결된 질문
- 검증 실패로 폐기된 가설
- 현재 스프린트와 무관한 아이디어

## 정리 루틴
1. `doc/context-inbox.md`를 위에서 아래로 읽는다.
2. “유효/이관/삭제”로 라벨링한다.
3. 유효하지만 결정된 항목은 `doc/context-log.md`로 이동한다.
4. 다음 작업이 필요한 항목은 `doc/next-actions.md`에 옮긴다.
5. inbox에는 미해결 임시 항목만 남긴다.
