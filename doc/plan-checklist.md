# Repeato 요구사항 체크리스트 (project-context 정렬본)

> 기준 문서: `doc/project-context.md`
> 목적: 초기 컨텍스트를 기준으로 “이미 결정된 항목”과 “정의가 필요한 항목”을 분리해 실행 가능한 백로그를 유지한다.

---

## 0. 컨텍스트 스냅샷 (결정됨)
- 앱 목표: `Ebbinghaus forgetting curve` 기반 암기 앱
- 핵심 도메인: 영어 단어, 한자, 구구단
- 제품 축: 카드 UX, 스케줄링, 멀티 과제, Drive 동기화, 진행 모니터링, 결제/콘텐츠
- 브랜드 후보: 까먹지마 / 리핏토 / 깜빡이 / 한입복습 / 메모몽
- 현재 권장 이름: `Repeato` (대안: `Repitto`)

체크:
- [ ] 앱명 최종 확정 (`Repeato` vs `Repitto`)
- [ ] 앱 톤(친근/가벼움) UI 문구 가이드 확정

---

## 1. 카드형 암기 UX
결정됨:
- 카드 중심 학습 UX를 채택한다.
- 짧은 세션 기반 반복 학습 흐름을 기본으로 한다.

정의 필요:
- [ ] 카드 템플릿(영단어/한자/구구단) 필드 정의
- [ ] 정답 판정 방식(자가평가 vs 객관식/주관식) 1차 선택
- [ ] 오답 카드 재노출 규칙(즉시/지연/세션말미)
- [ ] 1세션 기본 길이(분), 세션당 신규 카드 상한

---

## 2. 망각곡선 기반 학습 스케줄
결정됨:
- `Ebbinghaus forgetting curve` 기반 스케줄을 핵심 로직으로 사용한다.

정의 필요:
- [ ] 스케줄 전략 선택(고정 간격 / 가변 / SM-2 유사)
- [ ] 초기 간격표 정의(예: 10m, 1d, 3d, 7d, 14d, 30d)
- [ ] 정답/헷갈림/오답 시 다음 간격 변화 규칙
- [ ] Overdue(연체) 카드 처리 정책
- [ ] 하루 복습량 상한 및 초과 이월 규칙

---

## 3. 동시 N개 학습 과제 (멀티 트랙)
결정됨:
- 동시 다중 과제 진행을 지원한다.
- 초기 예시는 “일상 영단어 + 구구단 100단 + 천자문”이다.

정의 필요:
- [ ] 무료/유료별 동시 활성 과제 수
- [ ] 과제별 목표(신규/복습 상한) 및 우선순위 정책
- [ ] 과제별 통계 vs 통합 대시보드 노출 기준
- [ ] 규칙형 과제(구구단) 생성 템플릿 명세

---

## 4. Google Drive 동기화
결정됨:
- Google Drive를 동기화 채널로 사용한다.

정의 필요:
- [ ] 동기화 범위(카드/기록/설정/구매상태) 확정
- [ ] 저장 포맷(JSON/SQLite) 및 파일 구조
- [ ] 충돌 해결 전략(Last Write Wins / 머지 / 수동 선택)
- [ ] 동기화 주기(수동/자동/백그라운드)
- [ ] 암호화/키 관리/복구 정책

---

## 5. 학습 진행 모니터링
결정됨:
- 진행 상황 모니터링 기능을 핵심 기능으로 포함한다.

정의 필요:
- [ ] 핵심 KPI 5개 선정
- [ ] “오늘 완료” 기준(복습 0건 또는 목표 달성) 정의
- [ ] 일/주/월 리포트 범위
- [ ] 과제별/통합 뷰 전환 정책
- [ ] 알림 정책(빈도, 시간대, DND)

---

## 6. 인앱 결제 + 학습 데이터 구매
결정됨:
- 학습 샘플 3일 사용 후 결제 모델을 적용한다.

정의 필요:
- [ ] 3일 시작 기준(앱 최초 실행 vs 샘플 최초 진입)
- [ ] 체험 종료 후 접근 정책(읽기 전용/잠금/일부 유지)
- [ ] 상품 모델(단품/구독/번들) 1차안
- [ ] 영수증 검증 전략(단말/서버)
- [ ] 구매 복원/환불/보류 상태 처리 플로우

---

## 7. 공통 기술 의사결정
정의 필요:
- [ ] 로컬 저장소 선택(SQLite/Realm/기타)
- [ ] 이벤트 로그 스키마(append-only 여부)
- [ ] 오프라인 우선 동작 범위
- [ ] 데이터 마이그레이션/백업/복원 정책

---

## 8. 단계별 출시 범위
MVP-1:
- 카드 UX + 기본 스케줄 + 과제 3개 + 로컬 저장 + 기본 통계

MVP-2:
- Drive 동기화 + 충돌 처리(단순) + 알림

MVP-3:
- 결제 + 3일 체험 + 콘텐츠 스토어

출시 전 체크:
- [ ] MVP-1 완료 기준 문장화
- [ ] MVP-2 진입 조건 정의
- [ ] MVP-3에서 필요한 서버 범위 확정

---

## 9. 실행 순서 (권장)
1. 카드 템플릿/정답 판정/세션 규칙 확정
2. 스케줄 알고리즘(간격표 + 레벨 + 연체 처리) 확정
3. 멀티 트랙 정책 확정
4. 통계 KPI/대시보드 정의
5. 동기화/결제 정책 확정
