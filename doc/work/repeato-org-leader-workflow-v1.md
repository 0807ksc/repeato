# Repeato Org Leader Workflow v1

## 메타
- 버전: v1
- 작성일: 2026-03-08
- 상태: Active
- 목적: 각 조직 리더가 task 관리와 진행 추적을 동일한 방식으로 분리 운영하게 한다.

## 1. 리더 공통 책임
- 자기 조직 task backlog 유지
- blocked/unblock 기준 명시
- 상위 조직(PM)에 escalation 필요 여부 판단
- 기능 산출물과 운영 산출물 분리 기록

## 2. 리더별 관리 항목

### PM Lead
- 작업 상태판
- 의사결정 항목
- 사용자 의사결정 요청 항목

### Design Lead
- UX 검토 대기 항목
- 사용자 피드백 반영 여부

### Engineering Lead
- 구현 범위
- 코드 리뷰 필요 항목
- 이슈/브랜치/PR 상태
- 테스트 커버리지 현황과 70% 미만 위험 항목

### QA Lead
- 테스트 시나리오
- 승인/반려 사유
- 커버리지 예외 승인 여부와 보완 일정

### Data/Domain Lead
- 도메인 모델 변경
- KPI/이벤트 스키마 영향

### Trust/Ops Lead
- 보안/운영 리스크
- 서버리스/배포 적합성
