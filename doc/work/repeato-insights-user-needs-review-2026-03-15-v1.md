## 2. 사용자 질문 재정의
- 내가 지금 하고 있는 `중2 초급 영어`는 전체에서 얼마나 했나?
- 오늘 목표는 얼마나 끝냈고, 아직 얼마나 남았나?
- 어떤 카드/유형이 특히 약한가?
- 다음 복습이 언제 몰려오나?
- 어제/지난주보다 나아지고 있나?
- 지금 무엇을 누르면 가장 도움이 되나?

## 3. 1차 검토
### 3.1 6개 항목 검토
- 앱 목적:
  - `Insights`는 숫자를 많이 보여주는 화면이 아니라 학습 상태를 해석해 주는 화면이어야 한다.
- 사용자 이익:
  - 사용자는 `전체 진행률`, `오늘 남은 양`, `약점 영역`을 빠르게 알고 싶다.
- 지속 가능 포인트:
  - 공부가 끝나지 않았다는 압박보다 `다음 한 걸음`을 제안해야 지속 사용으로 이어진다.
- 타깃 사용자:
  - 중2 초급 영어 학습자처럼 진도 감각이 약하고, 지금 어디쯤 왔는지 확인이 필요한 사용자.
- 필수 철학/핵심 기능:
  - 덱 단위 진행률과 오늘 세션 상태를 함께 보여줘야 한다.
- 금지 사항:
  - 랭킹, 점수 경쟁, 불안 유발형 적색 경고 남발.

### 3.2 1차 후보 항목
1. 현재 덱 전체 진행률
2. 오늘 목표 진행률
3. 남은 카드 수
4. 약한 카드/약한 유형
5. 다음 복습 몰림 시점
6. 최근 7일 학습 추세

### 3.3 1차 결론
- 기존 `오늘 완료율/정답률/오늘 상태` 3개만으로는 부족하다.
- 사용자가 말한 `중2 초급 영어 전체 진행률`은 `Insights` 첫 화면의 핵심 질문으로 승격한다.

## 4. 2차 검토
### 4.1 6개 항목 재검토
- 앱 목적:
  - `현재 덱 이해`와 `다음 행동 결정` 두 축으로 정리한다.
- 사용자 이익:
  - 덱 기준으로 얼마나 왔는지 보이면 학습이 막연하지 않다.
- 지속 가능 포인트:
  - `전체 중 얼마 했는지`와 `오늘은 어디까지 하면 되는지`를 분리해 보여줘야 과부하가 줄어든다.
- 타깃 사용자:
  - 회화 목표지만 현재는 단어 중심으로 학습하는 사용자.
- 필수 철학/핵심 기능:
  - `전체 진도`와 `오늘 진도`를 혼동 없이 분리 표기.
- 금지 사항:
  - 전체 카드 수만 강조해 압박을 주는 설계.

### 4.2 항목 정리
#### Must Have
1. 덱 전체 진행률
- 예: `중2 초급 영어 120장 중 46장 학습 경험`
- 이유: 사용자가 가장 직접적으로 궁금해하는 항목.

2. 오늘 목표 진행률
- 예: `오늘 30장 중 12장 완료`
- 이유: 오늘 학습 종료 기준을 알려준다.

3. 남은 복습량
- 예: `오늘 남은 카드 18장`, `내일 예정 9장`
- 이유: 지금 부담이 어느 정도인지 판단하게 해준다.

4. 약점 영역 요약
- 예: `여행 표현 약함`, `헷갈림 7장`, `모르겠음 4장`
- 이유: 사용자가 무엇을 다시 봐야 하는지 알게 해준다.

5. 바로 이어지는 CTA
- 예: `약점 다시 학습`, `오늘 이어서 학습`, `덱 상세 보기`
- 이유: 지표만 보고 멈추지 않게 한다.

#### Should Have
6. 최근 7일 추세
- 예: 학습한 카드 수, 정답률 추세
- 이유: 어제/지난주보다 나아졌는지 확인 가능.

7. 다음 복습 몰림 시점
- 예: `오늘 저녁 8장`, `내일 오전 12장`
- 이유: 망각곡선 앱다운 계획 감각을 준다.

8. 덱 내 숙련도 분포
- 예: `초기/반복/안정` 카드 비율
- 이유: 단순 완료율보다 기억 안정성을 보여준다.

#### Later
9. 태그/주제별 진행률
- 예: `여행 60%`, `일상 35%`
- 이유: 콘텐츠 선호와 연결되지만 v1 첫 화면에서는 과밀할 수 있다.

10. 예문 vs 단어 비중
- 이유: 현재 7:3 정책 점검용으로는 유의미하지만 초기 우선순위는 낮다.

### 4.3 2차 결론
- `Insights`는 `오늘 KPI 3개`에서 `덱 상태 + 오늘 상태 + 약점 + CTA` 구조로 확장돼야 한다.
- 첫 화면 최상단에는 `현재 학습 중인 덱 요약 카드`가 들어가야 한다.

## 5. 3차 검토
### 5.1 6개 항목 최종 검토
- 앱 목적:
  - 사용자가 `내가 어디쯤 왔는지`와 `지금 뭘 해야 하는지`를 동시에 이해하게 한다.
- 사용자 이익:
  - 전체 진도, 오늘 진도, 약점, 다음 복습 시점을 빠르게 파악한다.
- 지속 가능 포인트:
  - 막연한 죄책감 대신 작게 끝낼 수 있는 다음 행동을 제안한다.
- 타깃 사용자:
  - 짧은 시간에 공부하고, 전체 맥락을 보고 싶어하는 중2 초급 영어 사용자.
- 필수 철학/핵심 기능:
  - `덱 전체 진행률`은 `Insights` 핵심 항목으로 확정한다.
- 금지 사항:
  - 경쟁형 리더보드, 과도한 점수화, 과잉 분석.

## 6. 최종 결정
- 사용자가 요청한 `현재 중2 초급 영어 전체 진행률`은 `Insights`의 핵심 항목으로 확정한다.
- `Insights`는 `오늘 KPI 3개`만 보는 화면이 아니라 `현재 덱 상태 + 오늘 상태 + 약점 + 다음 행동`을 묶는 화면으로 재정의한다.
- v1 첫 화면은 아래 5개를 우선 구현 대상으로 본다.
  1. 현재 덱 전체 진행률
  2. 오늘 목표 진행률
  3. 오늘 남은 카드 수
  4. 약점 영역 요약
  5. CTA(`약점 다시 학습` 또는 `오늘 이어서 학습`)

## 7. 구현 전 확인 필요
1. `덱 전체 진행률`의 정의
- `한 번이라도 본 카드 비율`인지, `안정 구간에 들어간 카드 비율`인지 확정 필요

2. `오늘 남은 카드 수`의 정의
- `오늘 목표 기준 잔량`과 `실제 due 잔량`을 분리할지 확정 필요

3. `약점 영역`의 기준
- 카드 단위인지, 태그/주제 단위인지 확정 필요
