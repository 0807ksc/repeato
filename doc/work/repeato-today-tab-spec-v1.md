  - 세션 종료/재진입/연체 카드 상태가 모두 문서화됨

## 2. 기존 문서 통합 요약

### 2.1 기존에 이미 확정된 사항
- `Today`는 5탭 IA의 첫 탭이다.
- 목적은 "오늘 해야 할 복습을 가장 빠르게 완료"하는 것이다.
- 기본 세션은 짧고 반복적인 루프를 전제로 한다.
- 목표 단위는 시간보다 카드 수 중심이다.
- 오답 카드는 우선 재노출한다.
- 앱 진입 후 5초 내 학습 시작 가능해야 한다.

### 2.2 기존 문서의 빈칸
- `Today Home`에서 무엇을 보여주면 바로 시작할 수 있는지 부족하다.
- 세션 일시중단/이탈/재진입 규칙이 없다.
- "오늘 완료" 기준이 문장으로 확정되지 않았다.
- 연체 카드가 많을 때 어떤 방식으로 부담을 줄일지 정해지지 않았다.
- 세션 종료 후 다음 액션이 `Today`, `Decks`, `Insights` 어디로 이어지는지 약하다.

## 3. 참여자 논의 정리

### 3.1 PM 관점
- `Today`는 "무엇을 할지 고민하는 화면"이 아니라 "바로 시작하는 화면"이어야 한다.
- 사용자가 매일 들어와도 피로하지 않게, 목표는 한 번에 1개만 강조해야 한다.
- 오늘 목표와 완료 상태는 과장 없이 명확해야 한다.

### 3.2 UX 디자이너 관점
- 첫 화면에서 숫자가 많으면 부담이 커진다.
- `학습 시작` CTA는 하나로 유지하되, "왜 이 수만큼 해야 하는지"를 짧게 설명해야 한다.
- 완료 직전/완료 후 상태가 다르게 보여야 성취감과 다음 행동이 생긴다.

### 3.3 App 개발자 관점
- `Today`는 복잡한 설정 UI보다 큐 상태를 안정적으로 표현하는 쪽이 우선이다.
- 상태는 최소 `시작 전`, `진행 중`, `완료`, `연체 많음`, `비어 있음`, `오프라인/동기화 지연` 정도로 쪼개야 구현과 테스트가 쉽다.
- 세션 중 인터럽트가 생겼을 때 복구 규칙이 명확해야 데이터 오염을 막을 수 있다.

### 3.4 사용자 관점
- "지금 당장 몇 장만 하면 되는지"가 가장 중요하다.
- 길게 끌리면 시작 자체를 미루게 된다.
- 오늘 목표를 다 했는지, 어제보다 나아졌는지 정도만 빠르게 보고 싶다.

## 4. 사용자 확인 질문
- 기본 진입 CTA는 `학습 시작` 하나로 두고, `10분 추천` 같은 보조 문구만 붙여도 충분한가?
- 오늘 카드가 많을 때 `전체 수`보다 `지금 처리할 10장 묶음`을 먼저 보여주는 방식이 더 편한가?
- 세션 종료 후 바로 `계속 학습`을 누르기보다 `오늘 종료`를 더 쉽게 눌러야 부담이 줄어드는가?
- 오답 카드는 같은 세션 안에서 바로 다시 보는 편이 좋은가, 잠깐 뒤에 묶어서 보는 편이 좋은가?
- 완료 후 보고 싶은 정보는 무엇인가:
  - 완료 카드 수
  - 정답률
  - 다음 복습 예정 시각
  - 연속 학습 일수

## 5. Today 탭 상세 사양

### 5.1 제품 목적
- 사용자가 앱을 열면 가장 적은 판단으로 오늘 복습을 시작하고, 짧은 성공 경험을 쌓게 한다.

### 5.2 핵심 JTBD
1. "지금 짧게라도 복습을 끝내고 싶다."
2. "오늘 무엇을 해야 하는지 계산하지 않고 바로 시작하고 싶다."
3. "끝난 뒤에는 오늘 할 일을 충분히 했다는 확신을 받고 싶다."

### 5.3 정보 구조
1. Today Home
2. Session
3. Session Summary
4. Completed State

### 5.4 Today Home
목표:
- 사용자가 5초 안에 시작한다.

노출 정보:
- 오늘 남은 카드 수
- 지금 바로 처리할 추천 묶음 수
- 과제별 대기 카드 요약
- 한 줄 안내 문구
- 주 CTA `학습 시작`

표시 규칙:
- 최상단 숫자는 `오늘 남은 카드 수` 하나만 크게 보여준다.
- 보조 정보는 `지금 10장부터 시작` 같은 추천 묶음으로 축소한다.
- 과제별 요약은 최대 3개만 노출하고 나머지는 `외 N개 과제`로 묶는다.
- 연체 카드가 많은 날에도 경고형 톤 대신 "오늘은 가볍게 10장부터"처럼 부담을 낮추는 문구를 사용한다.

권장 문구 예시:
- 기본: `오늘 24장 남았어요`
- 추천: `지금은 10장부터 시작하는 게 좋아요`
- 완료 직전: `거의 끝났어요. 6장만 더 하면 오늘 완료예요`

### 5.5 Session
목표:
- 빠른 인출 연습과 단순한 자가판단을 지원한다.

카드 흐름:
1. 문제 앞면 노출
2. `뒤집기`
3. `알겠음` / `헷갈림` / `모르겠음`
4. 규칙에 따라 다음 카드 큐잉

세션 규칙:
- 기본 추천 묶음은 10장이다.
- 사용자는 세션 중간에 종료할 수 있다.
- 오답/헷갈림 카드는 같은 세션 후반부에 우선 재노출한다.
- 연속 오답 3회 이상 카드가 있으면 세션 요약에서 별도 안내한다.
- 세션 진행률은 `12/24`처럼 전체 대비로 표시하되, 실제 경험은 10장 묶음 단위로 끊어준다.

인터럽트 처리:
- 앱 백그라운드 전환 후 재진입 시 마지막 카드 위치를 복구한다.
- 정오답 입력 직후 앱이 종료돼도 마지막 확정 결과만 저장한다.
- 세션을 이탈해도 이미 처리한 카드는 다시 증가시키지 않는다.

### 5.6 Session Summary
목표:
- 사용자가 "충분히 했다"는 감각을 갖고, 다음 행동을 고를 수 있게 한다.

노출 정보:
- 이번 세션 완료 카드 수
- 정답률
- 다시 볼 카드 수
- 예상 다음 복습 시각

버튼 정책:
- 1순위 CTA: `계속 학습`
- 2순위 CTA: `오늘 종료`
- 단, 완료에 근접했을 때는 `오늘 완료까지 6장 더` 보조 문구를 함께 노출한다.

### 5.7 Completed State
목표:
- 완료 판정을 단순하고 신뢰 가능하게 전달한다.

오늘 완료 기준:
- `오늘 예정된 복습 카드가 0건`이거나
- `오늘 목표 카드 수를 달성`했고 남은 카드는 전부 미래 예정 카드인 경우

완료 후 노출 정보:
- 오늘 완료 문구
- 오늘 처리 카드 수
- 어제 대비 변화 1개
- 선택 CTA:
  - `가볍게 더 하기`
  - `Insights 보기`

### 5.8 예외 상태
1. 비어 있음
- 노출: `아직 학습할 카드가 없어요`
- CTA: `Add에서 첫 카드 만들기`

2. 연체 많음
- 노출: `밀린 카드가 있지만 오늘은 10장만 먼저 하면 됩니다`
- 정책: 전체 밀린 양보다 지금 처리 묶음을 강조한다.

3. 동기화 지연
- 노출: 마지막 동기화 시각만 조용히 노출한다.
- 원칙: 학습 진행 자체는 막지 않는다.

4. 오프라인
- 노출: `오프라인에서도 학습은 계속돼요`
- 원칙: 결과를 로컬에 저장하고 추후 동기화한다.

## 7. Education/Cognitive Science 검토

### 7.1 Education Specialist
- 오늘 목표는 학습자에게 "완주 가능"하게 느껴져야 한다.
- 과제별 대기량보다 완료 경험을 우선해야 중도 이탈이 줄어든다.
- 초급 학습자는 `헷갈림` 선택지를 두는 편이 자기평가 부담을 줄인다.

### 7.2 Cognitive Science Specialist
- 같은 세션 안의 짧은 지연 재노출은 인출 연습 유지에 유리하다.
- 연체 카드가 많을수록 전체 잔량을 정면으로 보여주기보다 작은 배치로 분절해야 회피 반응을 줄일 수 있다.
- 완료 판정은 "오늘 예정 복습 0건"과 "목표 달성"을 분리해 혼동을 줄이는 것이 좋다.

## 8. Today 탭 3차 회의체 검토

### 8.1 1차 검토
- 앱 목적: 즉시 시작 가능한 일일 복습 허브로 정의
- 사용자 이익: 고민 없이 시작, 짧은 성취, 다음 복습 예측 확보
- 지속 가능 포인트: 과도한 죄책감 대신 완주 가능성 강조
- 타깃 사용자: 짧은 시간 학습하는 학생/직장인
- 필수 철학/핵심 기능: 빠른 시작, 짧은 세션, 신뢰 가능한 완료 기준
- 금지 사항: 연체량 공포 조장, 설정 유도 과다, 세션 중 불필요한 이탈

### 8.2 2차 검토
- 합의: `오늘 남은 카드 수`와 `추천 묶음 수`를 분리 표기한다.
- 수정: 정오답 이분법 대신 `알겠음/헷갈림/모르겠음` 3단계로 조정한다.
- 보류: 장기적으로 음성/받아쓰기 진입 버튼을 `Today`에 둘지 여부는 후속 검증으로 넘긴다.

### 8.3 3차 검토
- 합의: `Today`는 학습 시작 허브이며 설정 허브가 아니다.
- 합의: 완료 기준은 문서 `5.7` 정의를 우선 채택한다.
- 합의: 연체/오답은 부담 감소형 메시지와 작은 배치 전략을 사용한다.
- 보류: 추천 배치 기본값 `10장`은 실제 사용자 인터뷰 후 조정 가능하다.

## 9. 상위 의사결정자 리드백 2회

### 9.1 1차 리드백
의견:
- `Today Home`에 숫자가 많아질 가능성이 있으니 우선순위를 더 강하게 잘라야 한다.
- 완료 기준은 `목표 달성`과 `예정 복습 0건` 두 가지가 섞여 혼란을 줄 수 있다.

반영:
- 최상단 핵심 숫자를 `오늘 남은 카드 수` 1개로 고정했다.
- 완료 기준 문구를 `완료`와 `권장 완료`로 UI에서 구분하도록 메모를 추가했다.

### 9.2 2차 리드백
의견:
- 세션 종료 화면이 다음 행동을 너무 많이 제시하면 사용자가 멈춘다.
- 연체가 많은 사용자에게는 "전체 부채"보다 "지금 할 최소 행동"이 보여야 한다.

반영:
- 세션 요약 CTA를 `계속 학습`과 `오늘 종료` 2개로 고정했다.
- 연체 많음 상태의 문구와 정책을 `10장 우선` 구조로 정리했다.

## 10. 최종 결정
- `Today` 탭은 "오늘의 복습 실행 화면"으로 고정한다.
- 첫 화면은 `오늘 남은 카드 수`와 `학습 시작`을 중심으로 최소 구성한다.
- 추천 묶음 기본값은 `10장`으로 두고 후속 인터뷰로 검증한다.
- 평가지표는 `알겠음/헷갈림/모르겠음` 3단계 자가평가를 기본안으로 둔다.
- 완료 판정은 `완료`와 `권장 완료`를 구분해 혼동을 줄인다.
