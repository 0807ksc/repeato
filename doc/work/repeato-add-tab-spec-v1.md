# Repeato Add Tab Spec v1

## 메타
- 버전: v1
- 작성일: 2026-03-08
- 상태: Active
- 범위: `Add` 탭 상세 기획, UX, 앱/서버 연동 검토, 검증 질문
- 통합 대상:
  - `doc/work/repeato-menu-and-screen-spec-v1.md`의 `3.3 Add`
  - `doc/reference/plan-checklist.md`의 카드 템플릿 관련 항목

## 1. 작업 프레임
- 기본 Agent 세트: `PM + Product Designer + Technical Architect + Frontend Engineer + QA Engineer + Education Specialist`
- 필요시 추가 Agent: `Backend Engineer`
- 추가 이유: 가져오기 포맷, 중복 처리, 서버 저장 형식과 직접 연결되기 때문

## 2. 참여자 논의 정리
- PM: `Add`는 빠르게 넣고 나중에 다듬는 흐름이어야 한다.
- UX 디자이너: 첫 진입에서 선택지가 많아도 각 행동 목적이 명확하면 괜찮다.
- App 개발자: 직접 입력과 대량 가져오기는 상태 관리가 달라서 초기에 분리해야 한다.
- 사용자: 카드 추가는 학습 흐름을 끊지 않게 짧아야 한다.

## 3. 사용자 확인 질문
- 가장 자주 쓸 입력 방식은 무엇인가:
  - 직접 입력
  - 템플릿 선택
  - CSV/텍스트 가져오기
- 카드를 저장한 직후 바로 `Today`로 돌아가는 흐름이 필요한가?
- 중복 카드가 발견되면 자동 병합보다 사용자 확인이 더 나은가?

## 4. 상세 사양

### 4.1 제품 목적
- 사용자가 학습 아이디어를 즉시 카드로 만들고, 최소 편집으로 학습 큐에 넣게 한다.

### 4.2 핵심 화면
1. Add Entry
2. Card Composer
3. Bulk Import Preview

### 4.3 Add Entry
- 진입 옵션:
  - `직접 입력`
  - `템플릿 선택`
  - `가져오기`
- 각 옵션 아래에 한 줄 설명을 둔다.

### 4.4 Card Composer
- 필수 필드: 앞면, 뒷면, 덱
- 선택 필드: 예문, 힌트, 태그
- 저장 후 선택 CTA:
  - `계속 추가`
  - `Today로 이동`

### 4.5 Bulk Import Preview
- 행별 오류, 중복, 누락 필드를 표시한다.
- 사용자는 전체 확정 전 오류 행만 제외할 수 있다.

## 5. 연동 질문
- `Decks` Agent에게 질문: 새 카드 저장 후 어느 덱 상세에 반영되는지 규칙을 맞출 것인가?
- `Today` Agent에게 질문: 새 카드가 저장된 당일 Today 큐에 바로 들어갈지, 다음 세션부터 반영할지 결정이 필요한가?
- `Profile` Agent에게 질문: 유료 콘텐츠/샘플 콘텐츠 추가 경로를 Add에 둘지 Profile/Store로 분리할지?
- `Backend Engineer` 확인 필요:
  - 가져오기 파일 포맷
  - 중복 판정 키
  - 대량 저장 시 충돌 처리 정책

## 6. 3차 검토 요약
1. 1차 검토
- 목적: 콘텐츠 입력 허브
- 금지: 긴 마법사형 플로우 강제

2. 2차 검토
- 합의: 직접 입력과 대량 가져오기 분리
- 수정: 저장 직후 `Today로 이동` 보조 CTA 추가

3. 3차 검토
- 합의: 필수 필드 최소화
- 보류: OCR/카메라 입력은 v1 제외

## 7. 상위 의사결정자 리드백 2회
1. 1차 리드백 반영
- 처음부터 고급 옵션을 열지 않고 선택 필드로 내렸다.

2. 2차 리드백 반영
- 저장 후 흐름을 `계속 추가`와 `Today로 이동` 2가지로 고정했다.

## 8. 최종 결정
- `Add`는 빠른 입력과 짧은 검증에 집중한다.
- 저장 후 학습으로 이어지는 복귀 경로를 명시한다.

## 9. 후속 작업
1. 카드 템플릿 필드 정의를 `doc/reference/plan-checklist.md`에 확정 반영한다.
2. 대량 가져오기 중복 규칙을 서버 저장 모델과 맞춘다.
