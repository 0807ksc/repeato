## 4. 사용자 그룹 피드백 시뮬레이션

### Group A. 짧은 시간 학습자
- 관찰:
  - 시작은 빠르다.
  - `헷갈림` 버튼이 있으면 억지로 정답/오답을 고르지 않아도 된다.
- 불편:
  - 상단 설명은 짧을수록 좋다.
- 요청:
  - 완료 직전 남은 카드 수를 더 강하게 보여달라.

### Group B. 초급 학습자
- 관찰:
  - `헷갈림`이 심리적 부담을 줄인다.
- 불편:
  - 스와이프 제스처와 버튼 의미를 동시에 학습해야 한다.
- 요청:
  - 첫 1~2회는 버튼 위주 안내가 더 필요하다.

### Group C. 정리 중심 사용자
- 관찰:
  - Decks/Add의 역할 설명은 이해되지만 아직 실제 조작이 없다.
- 불편:
  - 현재는 "될 예정" 정보가 많아 실제 구조 검증은 어렵다.
- 요청:
  - 다음 반복에서는 Add 직접 입력과 Deck 상세 진입을 실제로 보여달라.

### Group D. 신뢰 민감 사용자
- 관찰:
  - Profile에서 동기화/결제가 아직 미구현이라는 점은 명확하다.
- 불편:
  - 무엇이 로컬 저장되고 나중에 동기화될지 더 설명이 필요하다.
- 요청:
  - 로컬 저장/동기화 정책을 앱 안에서도 짧게 알려달라.

## 5. PM 의사결정
- 결정 1:
  - Today의 3단계 평가는 유지한다.
- 결정 2:
  - 다음 반복은 Add 직접 입력과 Deck 상세 진입의 실동작을 우선한다.
- 결정 3:
  - 동기화/결제 구현은 여전히 보류하고, 로컬 저장 설명을 사용자에게 더 명확히 제공한다.
