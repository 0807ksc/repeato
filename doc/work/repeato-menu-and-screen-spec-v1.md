# Repeato Menu and Screen Spec v1

## 메타
- 버전: v1
- 작성일: 2026-02-22
- 상태: Active
- 기준: iOS 기본 레이아웃 패턴(사용자 익숙성 우선)

## 1. 기본 레이아웃 원칙 (iPhone 표준)
- 하단 `Tab Bar` 5개 고정
- 각 탭은 상단 `Navigation Bar` + 스크롤 콘텐츠
- 리스트는 iOS 기본 `List` 패턴, 상세는 `Push` 전환
- 중요 액션은 우측 상단 `+` 또는 하단 CTA 버튼
- 시스템 컴포넌트 우선 사용(토글, 피커, 시트, 얼럿)

## 1.1 핵심 사용자 기준(Interview 001 반영)
- 사용자 수준: 중2 수준 영어
- 학습 목표: 회화 우선, 초기 단계는 단어 암기 중심
- 시간 제약: 준비 없이 바로 시작 가능한 짧은 세션(10분/30분)
- 콘텐츠 선호: 단어 70% + 예문 30%
- 주제 우선순위: 여행 > 일상 > 업무
- 언어 정책: 영어 중심, 난이도 초과 문장/단어는 한국어 혼합 허용
- 오답 정책: 망각곡선 기반 적당한 반복
- 주말 정책: 평일과 동일 목표 유지

## 2. 필수 메인 메뉴 (Tab Bar)
1. Today (오늘 복습)
2. Decks (학습 과제)
3. Add (콘텐츠 추가)
4. Insights (진도/분석)
5. Profile (설정/계정)

---

## 3. 메뉴별 주요 기능 화면

### 3.1 Today
목적:
- 오늘 해야 할 복습을 가장 빠르게 완료

핵심 화면:
1. Today Home
- 오늘 복습 카드 수
- 과제별 대기 카드 요약
- `학습 시작` CTA

2. Session Screen (카드 학습)
- 앞면(문제) / 뒤집기 / 정답·오답 버튼
- `다시 보기`/`보류` 액션
- 진행률(예: 12/40)

3. Session Summary
- 완료 카드 수, 정답률, 예상 다음 복습 시간
- `계속 학습` / `오늘 종료`

주요 기능:
- 망각곡선 기반 카드 자동 큐잉
- 오답 카드 우선 재노출
- 짧은 세션 반복(10분 기본, 30분 확장)
- 목표 단위는 시간보다 카드 수 중심

### 3.2 Decks
목적:
- 학습 과제(덱) 생성/관리

핵심 화면:
1. Deck List
- 덱 이름, 진행률, 다음 복습 예정
- 상태 필터(진행중/일시중지/완료)

2. Deck Detail
- 카드 수, 숙련도 분포
- `학습 시작`, `카드 관리`, `설정`

3. Deck Edit
- 덱명/카테고리/난이도
- 복습 정책(기본/고급)

주요 기능:
- 다중 과제 동시 운영
- 덱별 목표/일일 카드 수 설정
- 덱 단위 일시중지/재개

### 3.3 Add
목적:
- 학습 콘텐츠를 빠르게 추가

핵심 화면:
1. Add Entry
- `직접 입력`, `템플릿 선택`, `가져오기`(CSV/텍스트)

2. Card Composer
- 앞면/뒷면 입력
- 예문/힌트/태그 추가
- 저장 시 덱 선택

3. Bulk Import Preview
- 파싱 결과 미리보기
- 중복/오류 행 표시 후 확정

주요 기능:
- 단건/대량 입력 지원
- 태그 기반 분류
- 오류 데이터 검증 후 저장

### 3.4 Insights
목적:
- 학습 지속성과 기억 유지 상태를 확인

핵심 화면:
1. Dashboard
- 오늘 완료율, 7일 연속 학습, 기억 유지 점수

2. Weak Cards
- 오답률 높은 카드 리스트
- `약점 집중 학습` 시작

3. Trend
- 주/월 단위 학습량, 정답률 추이

주요 기능:
- 과제별 KPI 모니터링
- 취약 카드 자동 큐 생성
- 학습 습관/성과 시각화
- 어제 대비 개선 지표(완료 카드 수/정답률/오답 재회상률)

### 3.5 Profile
목적:
- 계정/동기화/결제/앱 설정 관리

핵심 화면:
1. Account & Sync
- 로그인 상태
- 동기화 상태/마지막 동기화 시각

2. Plan & Purchase
- 샘플 사용 잔여일
- 콘텐츠 패키지 구매

3. Preferences
- 알림 시간/빈도
- 학습 목표(일일 카드 수)
- 개인정보/보안 설정

주요 기능:
- 결제/구독 관리
- 동기화 제어
- 알림/개인화 설정
- 짬시간 기반 약한 리마인드 + 개인 학습시간 예측 알림

## 4. 공통 사용자 플로우
1. 첫 진입
- 온보딩 -> 샘플 덱 선택 -> Today 시작

2. 일일 사용
- Today 진입 -> Session 완료 -> Summary 확인 -> Insights 점검

3. 콘텐츠 확장
- Add로 카드/덱 추가 -> Decks에서 구조 정리 -> Today에 자동 반영

4. 전환/과금
- 샘플 기간 종료 전 Profile의 Plan 화면에서 결제

## 5. 필수 UX 규칙
- 한 화면의 1차 목표는 하나만 둔다.
- 학습 세션 중 불필요한 이탈 액션을 최소화한다.
- 알림은 학습 리마인드 목적만 사용하고 과도한 빈도는 금지한다.
- 핵심 KPI(완료율/정답률/연속학습)는 Today와 Insights에서 일관 표기한다.
- 앱 진입 후 5초 내 학습 시작 가능해야 한다(준비 단계 최소화).
- 콘텐츠 큐는 `단어 7 : 예문 3` 비율을 기본값으로 유지한다.
