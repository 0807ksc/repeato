# Repeato Insights Tab Spec v1

## 메타
- 버전: v1
- 작성일: 2026-03-08
- 상태: Active
- 범위: `Insights` 탭 상세 기획, UX, 앱/서버 연동 검토, 검증 질문
- 통합 대상:
  - `doc/work/repeato-menu-and-screen-spec-v1.md`의 `3.4 Insights`
  - `doc/plan-checklist.md`의 KPI/오늘 완료 기준 관련 항목

## 1. 작업 프레임
- 기본 Agent 세트: `PM + Product Designer + Technical Architect + Frontend Engineer + QA Engineer + Data Analyst`
- 필요시 추가 Agent: `Backend Engineer`
- 추가 이유: 이벤트 수집, 집계 주기, 기기 간 통합 통계가 서버와 연결될 수 있기 때문

## 2. 참여자 논의 정리
- PM: `Insights`는 죄책감 화면이 아니라 학습 지속성 화면이어야 한다.
- UX 디자이너: 지표가 많아지면 행동 유도가 사라지므로 핵심 3개부터 보여줘야 한다.
- App 개발자: 이벤트 스키마가 먼저 정해져야 화면이 안정적으로 구현된다.
- 사용자: 어제보다 나아졌는지, 오늘 끝났는지 정도를 가장 빨리 알고 싶다.
- 추가 사용자 피드백(2026-03-15): `현재 중2 초급 영어를 하고 있는데 해당 아이템의 전체 진행률을 보고 싶다.`

## 3. 사용자 확인 질문
- 가장 먼저 보고 싶은 지표는 무엇인가:
  - 오늘 완료율
  - 정답률
  - 연속 학습 일수
- 약점 카드 목록이 길 때 바로 학습으로 이어지는 버튼이 필요한가?
- 주간/월간 추이를 첫 화면에서 바로 보고 싶은가?

## 4. 상세 사양

### 4.1 제품 목적
- 사용자가 학습 지속성과 기억 유지 상태를 간단히 확인하고, 약점 학습으로 다시 연결되게 한다.

### 4.2 핵심 화면
1. Dashboard
2. Weak Cards
3. Trend

### 4.3 Dashboard
- 상단 현재 덱 카드:
  - 현재 학습 중인 덱명
  - 덱 전체 진행률
  - 오늘 남은 카드 수
- 핵심 KPI 3개:
  - 오늘 완료율
  - 정답률
  - 연속 학습 일수 또는 오늘 상태
- 보조 정보:
  - 어제 대비 변화 1개
  - 가장 약한 덱 1개

### 4.4 Weak Cards
- 오답률 높은 카드 목록
- CTA: `약점 집중 학습`

### 4.5 Trend
- 주/월 단위 학습량과 정답률 추이
- 기본값은 주간 보기

## 5. 연동 질문
- `Today` Agent에게 질문: 완료/권장 완료 상태를 Insights에서 어떻게 이어받을 것인가?
- `Decks` Agent에게 질문: 덱별 지표 drill-down 깊이를 어디까지 허용할 것인가?
- `Backend Engineer` 확인 필요:
  - 이벤트 저장 위치
  - 일/주/월 집계 시점
  - 기기 간 합산 기준

## 6. 3차 검토 요약
1. 1차 검토
- 목적: 진도와 기억 유지 확인
- 금지: 점수 경쟁형/불안 자극형 랭킹

2. 2차 검토
- 합의: KPI는 3개부터 시작
- 수정: 약점 카드 화면을 별도 분리

3. 3차 검토
- 합의: 인사이트는 분석보다 재행동 유도가 우선
- 보류: 배지/레벨업식 게임화는 제외

## 7. 상위 의사결정자 리드백 2회
1. 1차 리드백 반영
- 첫 화면 지표를 3개로 줄였다.

2. 2차 리드백 반영
- 각 지표에서 행동 연결점이 없으면 안 되므로 `약점 집중 학습` CTA를 강화했다.

## 8. 최종 결정
- `Insights`는 현재 학습 중인 덱의 진행 감각과 핵심 KPI, 약점 재학습 연결에 집중한다.
- 과한 분석 대신 다음 행동이 바로 보이게 설계한다.
- `덱 전체 진행률`과 `오늘 남은 카드`를 첫 화면의 핵심 항목으로 포함한다.
