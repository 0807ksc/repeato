# Repeato Profile Tab Spec v1

## 메타
- 버전: v1
- 작성일: 2026-03-08
- 상태: Active
- 범위: `Profile` 탭 상세 기획, UX, 앱/서버 연동 검토, 검증 질문
- 통합 대상:
  - `doc/work/repeato-menu-and-screen-spec-v1.md`의 `3.5 Profile`
  - `doc/reference/plan-checklist.md`의 동기화/결제/알림 관련 항목

## 1. 작업 프레임
- 기본 Agent 세트: `PM + Product Designer + Technical Architect + Frontend Engineer + QA Engineer + Security/Privacy`
- 필요시 추가 Agent: `Backend Engineer + DevOps/SRE`
- 추가 이유: 로그인, 동기화, 결제, 복원, 개인정보 정책이 서버/운영과 직접 연결되기 때문

## 2. 참여자 논의 정리
- PM: `Profile`은 설정의 끝이 아니라 신뢰 확인 화면이어야 한다.
- UX 디자이너: 결제/동기화/알림을 한 화면에 두되 우선순위를 분명히 해야 한다.
- App 개발자: 서버 범위가 확정되지 않으면 실제 화면 상태를 다 짜기 어렵다.
- 사용자: 동기화가 되고 있는지, 결제가 끝났는지, 알림이 언제 오는지가 중요하다.

## 3. 사용자 확인 질문
- `Profile` 첫 화면에서 가장 먼저 보고 싶은 것은 무엇인가:
  - 동기화 상태
  - 이용권/체험 상태
  - 알림 설정
- 샘플 기간 종료가 가까울 때 `Profile`에서만 보여도 충분한가?
- 로그인 전에도 로컬 학습을 계속하고 나중에 연결하는 방식이 편한가?

## 4. 상세 사양

### 4.1 제품 목적
- 사용자가 계정, 동기화, 체험/결제, 알림을 신뢰감 있게 관리하게 한다.

### 4.2 핵심 화면
1. Account & Sync
2. Plan & Purchase
3. Preferences

### 4.3 Account & Sync
- 로그인 상태
- 마지막 동기화 시각
- 충돌/오류 상태
- CTA:
  - `지금 동기화`
  - `계정 연결`

### 4.4 Plan & Purchase
- 샘플 사용 잔여일
- 구매 상태
- 복원 CTA

### 4.5 Preferences
- 알림 시간/빈도
- 일일 목표 카드 수
- 개인정보/보안 관련 링크

## 5. 연동 질문
- `Today` Agent에게 질문: 알림 설정과 Today 시작 추천 시점을 어떻게 연결할 것인가?
- `Add` Agent에게 질문: 콘텐츠 구매 후 신규 카드/덱을 어디서 보여줄 것인가?
- `Backend Engineer` 확인 필요:
  - 로그인/동기화 범위
  - 결제 영수증 검증 위치
  - 복원/환불/보류 상태 처리
- `Security/Privacy` 확인 필요:
  - 최소 수집 데이터
  - 로컬 저장 암호화 범위

## 6. 3차 검토 요약
1. 1차 검토
- 목적: 계정/동기화/과금 신뢰 허브
- 금지: 다크패턴형 과금 유도

2. 2차 검토
- 합의: 동기화 상태와 체험/결제 상태를 상단 우선 노출
- 수정: 알림은 약한 리마인드 원칙을 유지

3. 3차 검토
- 합의: 로그인 전 로컬 학습 허용 방향 유지
- 보류: 서버 구조 확정 전 구매 복원 상세 플로우는 placeholder 정의로 둔다

## 7. 상위 의사결정자 리드백 2회
1. 1차 리드백 반영
- `Profile`을 설정 모음이 아니라 신뢰 정보 우선 구조로 재정렬했다.

2. 2차 리드백 반영
- 결제 CTA보다 체험/상태 설명을 먼저 배치하도록 수정했다.

## 8. 최종 결정
- `Profile`은 동기화, 결제, 알림을 신뢰 중심으로 관리하는 화면으로 정의한다.
- 서버 범위 미확정 항목은 placeholder 상태와 질문 목록을 남겨 구현 리스크를 분리한다.

## 9. 후속 작업
1. 서버 범위 확정 후 동기화/결제 상태 머신을 상세화한다.
2. 보안/개인정보 체크리스트와 UI 문구를 맞춘다.
