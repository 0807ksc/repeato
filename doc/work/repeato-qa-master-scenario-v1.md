## 3. E2E 마스터 시나리오

### 시나리오 A. 첫 실행 후 첫 학습
1. 앱 실행
2. Today 진입
3. 오늘 남은 카드 수 확인
4. 학습 시작
5. 카드 뒤집기
6. `알겠음/헷갈림/모르겠음` 중 하나 선택
7. 세션 요약 확인
8. 종료 또는 계속 학습

기대 결과:
- 5초 내 시작 가능
- 진행률이 증가
- 요약 화면에서 다음 행동이 명확

### 시나리오 B. 덱 구조 확인 후 복습 재진입
1. Decks 진입
2. 오늘 해야 할 덱 확인
3. 특정 덱 상세 진입
4. `학습 시작` 탭
5. Today 세션으로 이동

기대 결과:
- Decks 정렬과 Today 상단 덱 요약 기준이 일치
- 덱 상세에서 Today로 자연스럽게 연결

### 시나리오 C. 카드 추가 후 당일 학습 반영
1. Add 진입
2. 카드 1장 입력
3. 저장
4. `Today로 이동`
5. Today 재진입

기대 결과:
- 저장 성공 상태가 분명
- 현재 세션이 아닌 새 진입 기준으로 큐 갱신

### 시나리오 D. 오늘 완료 후 Insights 확인
1. Today에서 목표량 달성
2. 완료/권장 완료 상태 확인
3. Insights 진입
4. 오늘 완료율/정답률/연속 학습 일수 확인

기대 결과:
- Today와 Insights의 완료 정의가 모순되지 않음

### 시나리오 E. 목표 카드 수 변경
1. Profile 진입
2. 일일 목표 카드 수 변경
3. Today 재진입

기대 결과:
- Today 목표 값이 Profile 설정과 동일

### 시나리오 F. 덱 일시중지
1. Decks에서 덱 하나 일시중지
2. Today 진입
3. Insights 진입

기대 결과:
- 중지한 덱은 Today 큐에서 빠짐
- 완료율 분모에서 제외됨

### 시나리오 G. 오프라인 학습
1. 오프라인 상태 진입
2. Today 학습 진행
3. Profile에서 동기화 지연 상태 확인

기대 결과:
- 학습 차단 없음
- 동기화 상태는 신뢰 정보로만 노출

## 4. 테스트 매트릭스
- 기능: Today 루프
  - 우선순위: P0
  - 현재 상태: 부분 구현
  - 결과: 기본 루프 실동작 확인
- 기능: Decks
  - 우선순위: P1
  - 현재 상태: 기획 중심, 구현 미완료
  - 결과: 시뮬레이션만 가능
- 기능: Add
  - 우선순위: P1
  - 현재 상태: 플레이스홀더
  - 결과: 시뮬레이션만 가능
- 기능: Insights

  - 우선순위: P2
  - 현재 상태: 플레이스홀더
  - 결과: 시뮬레이션만 가능

## 5. 실제 수행 결과

### 5.1 자동 테스트
- 실행일: 2026-03-08
- 명령:
  - `flutter test`
  - `flutter analyze`

결과:
- `flutter test`: 통과
- `flutter analyze`: 이슈 없음

커버된 항목:
- 앱 셸 로드
- Today 탭 노출
- `다시 보기` 탭 시 진행률 증가
- 카드 좌우 스와이프 시 진행률 증가

### 5.2 문서 기반 시뮬레이션
- 시뮬레이션 범위:
  - 탭 간 이동
  - Decks/Add/Profile/Insights 상세 플로우
  - 오프라인/동기화/완료율 정합성

판정:
- Today 중심 기본 흐름은 성립
- 교차 탭 정합성은 공통 상태 모델 확정 전까지 잠정 합의 상태
- Decks/Add/Profile은 실제 UI가 없어 문서 시뮬레이션까지만 수행

## 6. 주요 결함/리스크
1. Today 실제 구현은 `알겠음/헷갈림/모르겠음` 3단계가 아니라 `known/again` 2단계 구조다.
2. Decks/Add/Profile은 기획 문서만 있고 실제 앱 동작이 없다.
3. Insights는 세션 통계 일부만 반영하고 있어 완료율/덱별/주간 지표 검증이 불가능하다.
4. 로컬 저장소가 아직 없어서 앱 재실행/다중 디바이스/오프라인 누적 테스트를 할 수 없다.

## 7. QA 승인 판단
- 현재 승인 범위:
  - Today MVP 기본 학습 루프
- 현재 미승인 범위:
  - 전체 탭 통합 시나리오
  - 동기화/오프라인/복원 시나리오
  - KPI 기반 Insights 시나리오

4. Deck 일시중지/재개 테스트
