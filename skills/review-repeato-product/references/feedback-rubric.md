# Repeato Product Feedback Rubric

## 목적
- 제품을 기동/사용한 뒤 관찰을 요구사항으로 변환할 때 일관된 분류 기준을 제공한다.

## 분류
- `broken`:
  - 기대한 흐름이 실제로 작동하지 않음
- `missing`:
  - 핵심 사용자 가치에 필요한 기능이 없음
- `unclear`:
  - 문구/구조/상태 설명이 모호함
- `polish`:
  - 구현은 됐지만 사용자 노출 품질이 낮음
- `non-goal`:
  - 현재 버전 범위를 벗어남

## 요구사항으로 바꾸는 방법
1. 관찰을 한 문장으로 적는다.
2. 왜 사용자 가치가 막히는지 적는다.
3. 최소 요구사항을 2~4개로 자른다.
4. `doc/work/repeato-delivery-organization-v1.md` 기준 태그/조직을 붙인다.

## 자동 진행 기준
- 바로 진행 가능:
  - 문서 보강
  - 테스트 보강
  - 리뷰용 copy 제거
  - next-actions 등록
- 사용자 확인 필요:
  - 제품 범위 확장
  - 서버/결제/외부 연동
  - 데이터 모델 대수술
  - 우선순위 재조정이 큰 경우
